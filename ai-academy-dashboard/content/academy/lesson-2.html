2 Why AI Security Matters

🚨 Security is the #1 Issue

Security is the main driver for AI adoption — AND the main blocker. Companies that can't trust AI won't deploy it. Companies that trust AI without verification will regret it.

Your role: Build AI systems that clients can trust. Today we learn how to identify and fix security vulnerabilities.

Two Sides of AI Security

AI Security Drivers
📈

AI Enables Security

  • Automated threat detection
  • Data analytics for compliance
  • Faster vulnerability scanning
  • Continuous monitoring at scale
🚫

AI Creates Risks

  • Prompt injection attacks
  • Data leakage through models
  • Insecure integrations
  • Uncontrolled agent behavior
⚠

Never put confidential information in AI tools without verification. Even email addresses can be problematic if combined with other data.

5 Key Parts of an AI Agent Codebase

📁 .github/workflows/ ← CI/CD configuration (GitHub Actions)
📁 e2e/ ← End-to-end tests (Playwright/Cypress)
📁 public/ ← Static assets, patterns, frameworks
📁 scripts/ ← Utility scripts (seeding, migrations)
📁 src/ ← Main source code
📁 app/ ← Pages and routes
📁 components/ ← Reusable UI pieces
📁 hooks/ ← ⚠ Connectors (security-sensitive!)
📁 lib/ ← Shared utilities
📁 types/ ← TypeScript definitions
📄 .env ← 🚨 Environment variables (NEVER commit!)
📄 .env.example ← Template for environment variables
⚠

Hooks are where security issues often occur. They connect your app to external systems — databases, APIs, authentication. Always review them carefully.

4 OWASP Top 10 for LLM Applications

🔒 The Security Standard for AI

OWASP (Open Web Application Security Project) created a Top 10 specifically for LLM applications. These are the most critical security risks you must know.

LLM01

Prompt Injection

Attacker manipulates LLM through crafted inputs to bypass instructions or execute unintended actions.

LLM02

Insecure Output Handling

Insufficient validation of LLM outputs before passing to other systems (XSS, SSRF, code execution).

LLM03

Training Data Poisoning

Manipulation of training data to introduce vulnerabilities, biases, or backdoors.

LLM04

Model Denial of Service

Resource-heavy operations that cause service degradation or high costs.

LLM05

Supply Chain Vulnerabilities

Compromised components, models, or plugins from third parties.

LLM06

Sensitive Information Disclosure

LLM reveals confidential data, PII, or proprietary information in responses.

LLM07

Insecure Plugin Design

Plugins with excessive permissions, lack of input validation, or insecure integrations.

LLM08

Excessive Agency

LLM granted too much autonomy to take actions without human oversight.

LLM09

Overreliance

Users trusting LLM outputs without verification, leading to misinformation or security gaps.

LLM10

Model Theft

Unauthorized access to proprietary LLM models, weights, or configurations.

📚

Deep Dive: Visit OWASP LLM Top 10 for detailed descriptions, attack scenarios, and prevention strategies.

5 Security Code Analysis with AI

🔍 Using AI to Find Vulnerabilities

Your AI Tutor can analyze code for security issues. This is how enterprise security teams work — AI-assisted code review finds vulnerabilities faster than manual inspection.

🤖 Security Analysis Prompt

Analyze this codebase for security vulnerabilities. Focus on: 1. **OWASP LLM Top 10** - Prompt injection, insecure outputs, excessive agency 2. **Authentication/Authorization** - Are there proper access controls? 3. **Environment Variables** - Are secrets properly handled? 4. **Input Validation** - Is user input sanitized? 5. **API Security** - Are endpoints protected? 6. **Data Exposure** - Could sensitive data leak? For each issue found: - Severity: CRITICAL / HIGH / MEDIUM / LOW - Location: File and line number - Description: What's the vulnerability - Fix: How to remediate Start with the most critical issues. Here is the code: [PASTE CODE OR LINK TO REPO]

Common Vulnerabilities to Look For

🚨
Hardcoded Secrets

API keys, passwords, or tokens directly in code instead of environment variables.

🚨
Missing Authentication

API endpoints or pages accessible without proper authentication checks.

⚠
Unvalidated User Input

User input passed directly to LLM or database without sanitization.

⚠
Excessive Permissions

Database connections or API calls with more access than needed.

✅
Proper Error Handling

Errors are caught and don't expose sensitive information in responses.
